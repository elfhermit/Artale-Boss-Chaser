## 2024-03-22 - Custom Inputs Break Keyboard Navigation
**Learning:** In this application, custom inputs like `.drop-chip` and `.toggle-switch` relied on `display: none` or fully invisible states (`opacity: 0; width: 0; height: 0;`) for their hidden checkboxes. This completely breaks keyboard navigation because the browser cannot focus on an element with `display: none` or zero dimensions. Furthermore, even if focusable, the custom visual elements (`.chip-content`, `.slider`) lacked any `:focus-visible` styling, leaving keyboard users completely blind as to where their focus was.
**Action:** Always ensure that hidden inputs underlying custom UI controls are visually hidden but remain focusable (e.g., `position: absolute; opacity: 0; width: 1px; height: 1px; overflow: hidden; clip: rect(0,0,0,0);`). Additionally, explicitly map the `:focus-visible` state of the hidden input to the visual style of its adjacent custom sibling (e.g., `input:focus-visible + .chip-content { outline: 2px solid var(--color-primary); }`).
