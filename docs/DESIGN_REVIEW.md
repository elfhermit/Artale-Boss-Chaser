
# UI/UX Design Review & Optimization Plan

## 目標受眾分析：MMORPG 玩家 (Boss 獵人)
**特質**：
1.  **效率至上**：需要最快速度紀錄擊殺，分秒必爭。
2.  **視覺直覺**：一眼就要知道「現在該打誰」、「下一個是誰」。
3.  **多工環境**：常邊打怪邊看網頁，介面需支援「常駐」、「緊湊」或「手機查閱」。
4.  **長期紀錄**：需要保存數天甚至數週的資料。

## 現有介面評估

| 項目 | 評分 (1-5) | 優點 | 改進空間 |
|------|-----------|------|----------|
| **即時性** | 4.5 | Quick 按鈕非常方便，倒數計時清晰。 | 缺乏「倒數結束前」的強烈提醒 (如音效)。 |
| **易讀性** | 4.0 | 卡片式設計資訊清楚，顏色區分狀態。 | 卡片較佔空間，若 Boss 數量多 (40+) 需一直捲動。 |
| **操作效率** | 4.5 | 支援鍵盤快速鍵與批次輸入。 | 手機版操作可能需要太多捲動。 |
| **美觀度** | 4.0 | 深色模式符合玩家習慣。 | 缺乏遊戲主題的沈浸感 (如背景圖、特效)。 |

## 優化規劃與功能推薦

### 1. 視覺與互動體驗 (UX Improvements)

*   **[推薦] 緊湊清單模式 (Compact View)**
    *   **痛點**：目前卡片佔據版面大，一次只能看 6-8 隻王。
    *   **解法**：增加一個「切換檢視」按鈕，改為類似 Excel 的條列式清單，只顯示「名稱、狀態、倒數、頻道」。
    
*   **[推薦] 聲音/桌面通知 (Auditory Feedback)**
    *   **痛點**：玩家掛網或切換視窗時，容易錯過重生。
    *   **解法**：當時間 `< 2分鐘` 時播放 "Ding" 音效，或使用瀏覽器 Notification API 發送桌面通知。

*   **[優化] 智慧排序 (Smart Sorting)**
    *   **痛點**：預設照 ID 排序，玩家更關心「誰快出了」。
    *   **解法**：預設改為「依照重生時間排序 (即將重生 -> 等待中 -> 剛死)」，讓玩家打開網頁第一眼就看到目標。

### 2. 資料安全與同步 (Data & Sync)

*   **[必備] 資料匯出/匯入 (JSON Backup)**
    *   **痛點**：換電腦或瀏覽器清除快取，資料全失。
    *   **解法**：新增「備份資料」與「還原資料」按鈕，產生 JSON 檔案供玩家保存。

*   **[進階] 隊伍同步 (Party Sync)** (需後端)
    *   **痛點**：這類工具通常是 2-6 人隊伍一起用，目前單機版無法看隊友紀錄。
    *   **解法**：未來可整合 Firebase 或 WebSocket，實現「一人紀錄，全隊同步」。

### 3. 用戶介面細節 (UI Polish)

*   **地圖資訊整合**：滑鼠停留在 Boss 卡片時，顯示出沒地圖的小視窗 (Tooltip)。
*   **掉落物清單**：點擊 Boss 名稱可查看該 Boss 的高價掉落物資訊。
