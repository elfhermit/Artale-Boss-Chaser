PART 01 · OVERVIEW

三大核心痛點

經過閱讀 README 功能清單、SPEC 資料規格、與實際 index.html 結構後，目前最影響玩家「友善體驗」的三大方向：

A
資料風險

所有紀錄只存在 LocalStorage，README 也明說「目前可透過開發者工具」備份 — 90% 的玩家不會用 DevTools，換瀏覽器即全失。

B
新手摩擦

第一次打開沒有任何引導，「獵殺專注模式」、「自動跳頻」、「星星=幸運頻道」這些獨特概念需要時間消化。

C
手機操作密度

歷史紀錄 7 欄表格在 375px 寬螢幕會橫向擠壓；卡片清單在 40+ 隻王時需大量捲動。

SUGGESTION · 01
01

資料備份 / 還原 UI

把現有的 DevTools 操作搬到設定面板，玩家一鍵匯出 JSON、拖曳檔案還原。

P0 · 必備資料安全

現狀

SPEC 與 README 中說明「建議定期將資料匯出（目前可透過開發者工具）」。對 MMORPG 玩家而言這完全不可行 — 一旦清快取，幾週擊殺紀錄全失。

建議

  • 「設定」icon 按鈕 加在右上角主題切換旁。
  • 面板內含 「匯出 JSON」、「匯入 JSON」(拖曳/點擊)、「清空所有紀錄」(現有)。
  • 匯出檔名自動化：artale-boss-2026-04-29.json
  • 匯入時偵測 schema 版本，並顯示 「將合併 N 筆 / 取代現有」 二選一。
實作成本： 純前端、約 80 行 JS。Blob + URL.createObjectURL 即可，無後端依賴。

02

首次使用引導

用 3 步互動式教學介紹「鎖定 Boss → Enter 紀錄 → 自動跳頻」核心循環。

P0 · 必備新手

現狀

「獵殺專注模式」、「自動跳頻」、「Enter 秒速紀錄」、「星星 = 幸運頻道」是這款工具的差異化亮點，但首次玩家無從得知 — README 雖有教學，但開了網頁不會回去看。

建議

  • 第一次無紀錄時，自動彈出 3 步驟引導 overlay。
  • 每步驟用 spotlight (打洞遮罩) 高亮對應 UI 元素，配 1 行說明 + 「下一步」。
  • 右上角永久保留 ? 圖示，可重看引導 + 鍵盤快速鍵。
  • 首次完成後，第一筆紀錄完成自動 toast：「太棒了！系統已自動跳到下一頻道，繼續按 Enter 即可秒速紀錄」

03

手機版歷史紀錄改卡片時間軸

7 欄表格在 375px 螢幕會橫向擠壓；改用卡片時間軸維持資訊密度。

P1 · 短期手機

現狀

<table> 在 index.html 是 7 欄（Boss、時間、頻道、掉寶、備註、預計重生、操作）。手機橫向滾動體驗差，玩家邊打怪邊查不友善。

建議

  • 桌面 ≥ 880px 維持表格。手機切換成 時間軸卡片。
  • 每張卡片：左側時間（mm:ss 大字）+ 右側 Boss 名 + 頻道 + 掉寶 chip。
  • 整張可橫向 swipe → 露出「編輯／刪除」（iOS 列表手勢慣例）。

04

緊湊清單模式視覺再優化

README 已有「切換檢視」功能，但清單模式仍可再進一步：對齊欄位 + 倒數視覺化。

P1 · 短期資訊密度

現狀

已支援 view-mode-btn（切換檢視）。但卡片內部資訊（狀態燈、倒數、頻道、最後紀錄）視覺層級偏平，掃讀慢。

建議

  • 清單模式：固定欄位（圖示 / 名稱 / 狀態燈 / 倒數 mm:ss / 最近頻道），等寬 mono 字體，垂直對齊。
  • 倒數附 細條進度條 — 視覺看出剩多少。
  • 一眼掃 12 隻王，比卡片省 60% 高度。

05

桌面通知 + 音效強化

玩家常切到遊戲視窗，音效不夠 — 用 Browser Notification API 跳系統通知。

P1 · 短期即時性

現狀

專案已有 sound-toggle-btn 音效。但玩家切到全螢幕遊戲後，網頁的音效會被忽略 / 蓋掉。

建議

  • 新增 「桌面通知」 選項（Notification API 需使用者授權）。
  • 2 分鐘預警時跳系統通知，標題：「紅寶王 · 即將重生」，內文：「最早於 1:42 後出現 / 上次頻道 142」。
  • 點擊通知 → 自動 focus 此頁，並鎖定該 Boss 進入「獵殺專注模式」。
  • 音效部分：可選 3 種音色（Ding / Bell / Drum），避免長時間使用聽到膩。

06

頻道輸入：最近輸入 + 智慧建議

1–3000 個頻道輸入框很空曠；補一排「最近用過 / 幸運頻道」chip。

P1 · 短期效率

現狀

頻道輸入支援 1–3000 + 自動 +1。但玩家若要跳到「上次出貨的頻道」，得手動打進去。

建議

  • 專注面板的頻道輸入框下方加一排 chip。
  • 顯示 最近 3 個 輸入過的頻道 + 該 Boss 的幸運頻道 (Top 2)。
  • 點 chip 直接填入。幸運頻道 chip 用金色邊框視覺區分。
  • 長按某 chip → 「設為起始頻道」。

07

幸運頻道熱度視覺化

把每隻 Boss 的「星星」紀錄畫成 1–3000 頻道的熱力圖，一眼看哪個頻道最容易出貨。

P2 · 體驗資料洞察

現狀

「星星 / 幸運頻道」目前只在歷史紀錄裡用 ★ 標示。玩家難以一眼看出「這隻王在 200~250 頻道特別容易出貨」這類規律。

建議

  • 專注面板裡，下方加一個 「頻道熱度」 摺疊區。
  • 1–3000 頻道用 30×100 格網格呈現（可選 100 格一段聚合）。
  • 顏色深淺對應「擊殺次數 / 掉寶率」，金色點代表星星紀錄。
  • 滑鼠 hover 顯示：「頻道 142 · 擊殺 8 次 · ★ 3 次（37%）」。
玩家視角： 這把工具從「紀錄器」升級為「策略工具」 — 找出個人運氣榜的關鍵差異化功能。

08

迷你常駐視窗 (Mini PiP)

玩家邊打怪邊看，需要極簡的「總是置頂」小視窗。用 Document Picture-in-Picture API。

P2 · 體驗多工

現狀

玩家通常邊玩遊戲邊看網頁，瀏覽器整視窗占空間。Chrome / Edge 已支援 documentPictureInPicture API，可彈出常駐迷你視窗。

建議

  • 右上角加 「常駐」icon (open_in_new)。
  • 點擊彈出 240×320 的浮動視窗，只顯示 「即將重生」狀態的 Boss 倒數列表。
  • 整窗 click 透傳到主頁 → 點 Boss 主視窗鎖定該王。
  • Fallback：不支援的瀏覽器 → 用 window.open 開普通小視窗，跨頁透過 BroadcastChannel 同步資料。

09

空狀態與無紀錄引導

第一次打開沒紀錄時，「今日戰況」全 0 + 歷史空表格 = 失敗感。

P2 · 體驗情感

現狀

無紀錄時：今日戰況全部 0、歷史紀錄空表格、常用 Boss 提示文字。視覺很空又無引導性。

建議

  • 無紀錄時，「今日戰況」直接隱藏，改顯示「歡迎」橫幅。
  • 歷史頁面空狀態 → 大圖示 + 引導：「還沒有任何紀錄。先回 Boss 列表選一隻王開始吧 →」
  • 常用 Boss 空狀態 → 動畫指引箭頭 (CSS) 指向某張卡片的 ☆。

鍵盤快速鍵 Cheatsheet

這是熟練玩家的工具 — 完整鍵盤操作可以大幅提升效率，但要先讓他們發現。

P3 · 拋光進階

建議

  • 按 ? 彈出快速鍵 cheatsheet (modal)。
  • 新增更多快速鍵：/ 跳到搜尋、L 切換清單模式、1-9 鎖定常用 Boss、N 下一個即將重生、Esc 取消鎖定。

  • 右下角永久顯示「按 ? 看快速鍵」hint，3 秒後淡出。
